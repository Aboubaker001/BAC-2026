# Contributing to BAC-2026 English Study Guide

We welcome and appreciate contributions from everyone! By contributing, you help us create the largest and most comprehensive free English learning resource for the Algerian Baccalaureate.

## How to Contribute

There are many ways to contribute to this project:

*   **Improve Existing Content:** Correct errors, clarify explanations, add more examples, or update information.
*   **Add New Content:** Create new lessons, exercises, vocabulary lists, or study materials.
*   **Translate Content:** Help translate materials into other languages (e.g., Arabic, French) to reach a wider audience.
*   **Review Pull Requests:** Provide feedback on contributions from other community members.
*   **Report Bugs or Suggest Features:** Help us improve the project by reporting issues or suggesting new functionalities.
*   **Spread the Word:** Share the project with your friends, classmates, and teachers!

## Contribution Workflow

1.  **Fork the Repository:** Start by forking the `Aboubaker001/BAC-2026` repository to your GitHub account.
2.  **Clone Your Fork:** Clone your forked repository to your local machine:
    ```bash
    git clone https://github.com/YOUR_USERNAME/BAC-2026.git
    ```
3.  **Create a New Branch:** Create a new branch for your changes. Use a descriptive name that reflects the nature of your contribution (e.g., `feature/add-grammar-lesson`, `fix/typo-in-readme`, `docs/update-roadmap`).
    ```bash
    git checkout -b your-branch-name
    ```
4.  **Make Your Changes:** Implement your changes or additions. Ensure your code and content adhere to the project's quality standards and naming conventions.
5.  **Commit Your Changes:** Commit your changes with a clear and concise commit message. Follow the [Commit Convention](#commit-convention) below.
    ```bash
    git commit -m "feat: Add new lesson on reported speech"
    ```
6.  **Push to Your Fork:** Push your changes to your forked repository on GitHub.
    ```bash
    git push origin your-branch-name
    ```
7.  **Create a Pull Request (PR):** Go to the original `Aboubaker001/BAC-2026` repository on GitHub and open a new Pull Request. Provide a detailed description of your changes, why they are necessary, and any relevant context. Link to any related issues.

## Branch Naming Conventions

Use the following prefixes for your branch names:

*   `feature/`: For new features or significant additions.
*   `fix/`: For bug fixes.
*   `docs/`: For documentation updates.
*   `refactor/`: For code refactoring that doesn't change external behavior.
*   `style/`: For changes that do not affect the meaning of the code (white-space, formatting, missing semi-colons, etc).
*   `chore/`: For routine tasks, maintenance, or build process changes.

## Commit Convention

We follow the [Conventional Commits](https://www.conventionalcommits.org/en/v1.0.0/) specification. Each commit message should be structured as follows:

```
<type>(<scope>): <description>

[optional body]

[optional footer(s)]
```

**Type:** Must be one of the following:

*   `feat`: A new feature
*   `fix`: A bug fix
*   `docs`: Documentation only changes
*   `style`: Changes that do not affect the meaning of the code (white-space, formatting, missing semi-colons, etc)
*   `refactor`: A code change that neither fixes a bug nor adds a feature
*   `perf`: A code change that improves performance
*   `test`: Adding missing tests or correcting existing tests
*   `chore`: Changes to the build process or auxiliary tools and libraries suchs as documentation generation

**Scope (optional):** The scope could be anything specifying the place of the commit change (e.g., `grammar`, `vocabulary`, `readme`, `unit-01`).

**Description:** A short, imperative, present tense description of the change.

**Example:**

```
feat(grammar): Add new lesson on reported speech

This commit introduces a comprehensive lesson on reported speech, including theory, examples, and exercises.
```
